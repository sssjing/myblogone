记忆系统与规划能力

杨景2025-05-27大约 10 分钟

# 记忆系统 (Memory)

一个真正的 AI Agent 不仅需要"思考"和"行动"，还需要"记忆"。记忆系统是 Agent 能够在多轮交互中保持上下文、从过去经验中学习、并在未来做出更好决策的关键基础设施。没有记忆的 Agent 就像一个每隔几分钟就失忆的人——它无法积累经验，也无法提供连贯的服务。

在认知科学中，人类的记忆系统被分为多个层次，而 AI Agent 的记忆设计深受这一分类的启发。下面我们从人类记忆的类比出发，逐步展开 Agent 记忆系统的各个层面。

# 人类记忆的类比

心理学家将人类记忆分为三个层次，这一分类为 AI Agent 的记忆架构提供了天然的设计蓝图：

  • 感觉记忆 (Sensory Memory)：对应 Agent 对原始输入的感知，类似于模型接收到的 embedding 向量。这是最短暂的记忆形式，持续时间极短，只保留原始刺激的粗略表征。
  • 短期/工作记忆 (Short-term / Working Memory)：对应 Agent 的上下文窗口（context window）。这是 Agent 当前正在处理的信息，容量有限，但可以被主动操作和推理。就像人类的工作记忆只能同时处理 7±2 个信息块一样，Agent 的上下文窗口也受到 token 数量的硬性限制。
  • 长期记忆 (Long-term Memory)：对应 Agent 通过外部向量数据库存储的历史经验。容量几乎无限，但需要通过检索（retrieval）才能被激活使用。

这一三层架构的核心思想是：Agent 的"智能"不仅来自于模型本身的推理能力，更来自于它如何有效地存储、检索和利用信息。

# 短期记忆

Agent 的短期记忆就是它的上下文窗口（context window）。这是 LLM 在单次推理中能够"看到"的全部信息，包括系统提示、对话历史、工具调用结果等。

上下文窗口的发展经历了快速的容量增长：

  • 早期 GPT-3.5 仅有 4K tokens 的窗口
  • GPT-4 将窗口扩展到 128K tokens
  • Google Gemini 1.5 Pro 更是达到了 1M tokens 的超长上下文

然而，窗口越大并不意味着问题就解决了。短期记忆面临的核心挑战包括：

  • 信息稀释：随着对话长度增长，关键信息可能被大量无关内容"稀释"，模型对早期重要细节的关注度下降。
  • 成本与延迟：更长的上下文意味着更高的计算成本和更慢的推理速度。
  • "大海捞针"问题：即使窗口足够大，模型也未必能从中准确检索到需要的信息。

因此，实际的 Agent 系统通常会采用上下文压缩策略，如摘要历史对话、只保留关键信息片段、或者使用滑动窗口机制。

# 长期记忆

长期记忆为 Agent 提供了"无限容量"的历史经验存储。其核心架构基于向量数据库（Vector Database），工作流程如下：

  1. 编码：将文本经验通过 embedding 模型转化为高维向量
  2. 存储：将向量及其元数据存入向量数据库
  3. 检索：当 Agent 需要相关信息时，通过向量相似度搜索找到最相关的记忆片段

常用的向量存储方案包括：

  • Pinecone：全托管的向量数据库服务，开箱即用
  • Weaviate：开源向量搜索引擎，支持混合检索
  • ChromaDB：轻量级嵌入式向量数据库，适合快速原型开发
  • FAISS：Meta 开源的高效向量检索库，适合大规模场景

在检索算法层面，Lilian Weng 在其博客中总结了多种 MIPS（Maximum Inner Product Search）算法：

  • LSH（Locality-Sensitive Hashing）：通过哈希函数将相似向量映射到同一桶中
  • ANNOY（Approximate Nearest Neighbors Oh Yeah）：基于随机投影树的近似最近邻算法
  • HNSW（Hierarchical Navigable Small World）：基于图结构的分层导航算法，检索速度快、精度高
  • FAISS：Facebook 的向量检索库，支持多种索引类型
  • ScaNN（Scalable Nearest Neighbors）：Google 提出的高效向量检索算法

# Write/Read/Reflection 模块

李宏毅教授在其课程笔记中提出了 Agent 记忆系统的三大核心模块，这一框架清晰地定义了记忆的生命周期：

Write（写入）——存储经验

Agent 在每次交互后，将发生的事情、采取的行动、获得的结果等信息写入长期记忆。关键的写入策略包括：

  • 记录事件的原始细节（what happened）
  • 记录行动的成功/失败结果（what worked, what failed）
  • 记录关键决策点和推理过程

Read（读取）——检索相关经验

当 Agent 面临新任务时，通过 RAG（Retrieval-Augmented Generation）机制从长期记忆中检索最相关的经验。这不仅仅是简单的相似度匹配，还需要考虑：

  • 时间衰减：近期的记忆通常比远期的更相关
  • 任务相关性：与当前任务类型相似的经验优先级更高
  • 成功经验优先：那些导致成功结果的经验更有参考价值

Reflection（反思）——经验升华

这是记忆系统中最有趣的部分。反思机制让 Agent 不只是存储原始经验，还能定期将经验总结为更高层次的洞察。就像人类会"复盘"一样，Agent 可以：

  • 将多个具体事件抽象为一般性规则
  • 识别反复出现的模式和规律
  • 更新自己的行为策略和偏好

# MemGPT 与 A-MEM

MemGPT（2023）提出了一种受操作系统虚拟内存启发的分层记忆管理方案。就像 OS 通过页面调度在有限的物理内存和无限的虚拟内存之间切换一样，MemGPT 在有限的上下文窗口和无限的外部存储之间实现了自动的信息调度。当上下文窗口满时，系统会自动将不重要的信息"换出"到外部存储，并在需要时将其"换入"。

A-MEM（Autonomous Memory）则探索了更进一步的方向——让 Agent 自主地组织和管理自己的记忆。A-MEM 不依赖预定义的存储结构，而是让 Agent 自己决定什么值得记住、如何组织记忆、以及何时更新已有记忆。这更接近人类记忆的自主性和灵活性。

# Stream Bench 的发现

Stream Bench 的研究揭示了一个关于记忆有效性的重要发现：

  • 正面经验（成功案例）对未来任务有显著帮助，能够有效提升 Agent 在相似场景中的表现
  • 负面经验（失败案例）对未来的帮助非常有限，甚至可能产生误导

这一发现提示我们，在设计 Agent 记忆系统时，应该对正面经验给予更高的存储权重和检索优先级。同时也表明，仅仅记录"做了什么"是不够的——记忆的价值在于那些"做对了什么"。

# 规划能力 (Planning)

规划能力是 AI Agent 的另一个核心能力维度。如果说记忆让 Agent 能够"从过去学习"，那么规划则让 Agent 能够"为未来做准备"。一个没有规划能力的 Agent 只能被动地对每一步的输入做出反应，而具备规划能力的 Agent 可以在执行之前就制定出完整的行动方案。

# 什么是规划

规划是指 Agent 在执行任务之前，预先列出为达成目标所需的一系列行动步骤的过程。一个好的规划需要考虑：

  • 最终目标是什么
  • 当前状态与目标状态之间的差距
  • 可用的工具和资源
  • 可能的约束条件和限制
  • 潜在的风险和备选方案

值得注意的是，规划并非一成不变。在实际执行过程中，Agent 可能会遇到意外情况，需要动态调整计划（dynamic replanning）。这种"计划-执行-调整"的循环是 Agent 灵活性的重要体现。

# Chain of Thought (CoT)

Chain of Thought（思维链）是最基础也最重要的规划方法。其核心思想极其简单——"让我们一步步思考"（"Let's think step by step"）。

通过在 prompt 中引导模型逐步推理，CoT 显著提升了 LLM 在复杂任务上的表现。它的工作原理是：

  1. 将复杂问题分解为一系列更小的推理步骤
  2. 每个步骤的输出作为下一步的输入
  3. 逐步逼近最终答案

CoT 的优势在于简单、通用、不需要额外的基础设施。但它的局限也很明显：它是一条线性的推理路径，如果中间某一步出错，后续所有步骤都会受到影响。

# Tree of Thoughts (ToT)

Tree of Thoughts（思维树）是 CoT 的增强版本。与 CoT 的线性推理不同，ToT 允许 Agent 同时探索多条推理路径，就像在一棵树上进行搜索一样。

ToT 的工作流程如下：

  1. 生成候选思路：在每一步，生成多个可能的推理方向
  2. 评估：使用 LLM 自身或启发式方法评估每个方向的前景
  3. 搜索：使用 BFS（广度优先）或 DFS（深度优先）策略遍历推理树
  4. 回溯：当某条路径看起来没有前途时，回退到之前的节点尝试其他方向

这种"探索-评估-回溯"的机制使得 ToT 在需要创造性思维或复杂推理的任务上（如数学证明、创意写作、策略规划）表现出色。当然，代价是更多的 LLM 调用和更高的计算成本。

# Plan-and-Execute 模式

Plan-and-Execute 是一种更结构化的 Agent 架构模式，它将规划和执行明确分离为两个阶段：

  1. Plan 阶段：首先由一个"规划者"LLM 生成完整的行动计划，包括所有需要执行的步骤
  2. Execute 阶段：然后由一个"执行者"Agent 逐步执行计划中的每个步骤
  3. Re-plan 阶段：当执行过程中遇到预期之外的结果时，触发重新规划

与纯 ReAct 模式相比，Plan-and-Execute 的优势在于：

  • 全局视野：规划者可以看到整个任务的全貌，而不仅仅是当前一步
  • 效率更高：减少了不必要的试错步骤
  • 可解释性更强：用户可以查看和修改 Agent 的计划

典型的实现方式是使用一个强模型（如 GPT-4）作为规划者，一个快模型（如 GPT-3.5）作为执行者，在成本和能力之间取得平衡。

# LLM 的规划能力评估

LLM 究竟有多强的规划能力？这个问题一直是学术界争论的焦点。李宏毅教授在课程中引用了多项关键研究：

PlanBench（2023）

PlanBench 是第一个系统性评估 LLM 规划能力的基准测试。研究发现：

  • LLM 确实具备一定的规划能力，但这种能力有限且不稳定
  • 在简单的规划任务上，LLM 表现不错
  • 但随着任务复杂度增加（更多约束、更多步骤），性能迅速下降

TravelPlanner（2024）

TravelPlanner 专注于评估 LLM 在复杂约束条件下的规划能力。旅行规划任务涉及多种约束（时间、预算、偏好、交通连接等），研究发现：

  • 基于约束的复杂规划对 LLM 来说仍然极具挑战性
  • 即使是最先进的模型，在满足所有约束条件的同时生成可行方案方面仍然困难

正如李宏毅教授所总结的：LLM 的规划能力"介于有和无之间"——它们确实能做一定程度的规划，但远远不能称为可靠。

# 优化规划能力的方法

既然 LLM 原生的规划能力有限，研究者们提出了多种优化策略：

1. 暴力搜索（Brute-force Search）

最直接的方法是让 LLM 生成所有可能的计划，然后逐一评估。理论上能找到最优解，但计算成本呈指数增长，实际中几乎不可行。

2. 带剪枝的树搜索（Tree Search with Pruning）

借鉴棋类 AI 的思路，使用树搜索配合剪枝策略。李宏毅教授将这一过程形象地比喻为"脑内小剧场"——Agent 在心中模拟不同行动的可能结果，提前排除明显不好的选项，只对有前途的路径进行深入探索。

3. LLM 作为世界模型

让 LLM 充当"世界模型"，预测每个行动可能导致的状态变化。Agent 可以在这个"内心世界"中进行模拟和推演，而不需要在真实环境中试错。

4. 推理模型（Reasoning Models）

以 DeepSeek-R1 为代表的新一代推理模型，通过强化学习训练出更强的内在推理能力。这类模型在规划任务上展现出了显著的优势，它们能够进行更长链的推理、更好的自我纠错、以及更准确的约束满足。

5. 解决"过度思考"问题

研究发现，LLM 有时会"想太多"——在简单的任务上花费过多的推理步骤，反而导致性能下降。优化方向包括：动态调整推理深度、在简单步骤上"快速通过"、以及在困难步骤上"深入思考"。
