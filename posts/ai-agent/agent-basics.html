AI Agent 基础概念

杨景 2025-05-27 大约 8 分钟

什么是 AI Agent ​

在人工智能领域，Agent（智能体）是一个能够感知环境并采取行动以实现特定目标的实体。与传统的聊天机器人不同，AI Agent 具备自主决策能力，能够使用工具、进行多步推理，并在遇到问题时自我调整和迭代。

OpenAI 研究科学家 Lilian Weng 在其经典博文 "LLM Powered Autonomous Agents" 中提出了一个极具影响力的公式：

Agent = LLM + Planning（规划）+ Memory（记忆）+ Tool Use（工具使用）

这个公式清晰地勾勒出了现代 AI Agent 的核心构成。大语言模型（LLM）充当 Agent 的"大脑"，负责理解任务、推理和生成；规划能力使 Agent 能够将复杂任务分解为可执行的子步骤；记忆系统让 Agent 能够保留上下文信息并从过去的经验中学习；工具使用则赋予 Agent 与外部世界交互的能力——无论是调用 API、执行代码还是操控浏览器。

与普通聊天机器人的关键区别在于：

  • 自主性（Autonomy）：Agent 可以自主决定下一步该做什么，而非被动地回答问题。
  • 工具使用（Tool Use）：Agent 能够调用外部工具和 API 来完成任务，突破纯文本生成的限制。
  • 决策与迭代（Decision-making & Iteration）：Agent 可以根据中间结果调整策略，进行多轮推理和自我纠错。
  • 目标导向（Goal-oriented）：Agent 以完成用户指定的最终目标为导向，而非仅仅生成一段回复。

Agent 的发展历程 ​

AI Agent 的概念虽然由来已久，但真正引爆这一领域的契机是大语言模型能力的飞跃。以下是近年来 Agent 领域的关键里程碑：

2023 年：Agent 元年 ​

2023 年初，AutoGPT 和 BabyAGI 横空出世，首次向大众展示了 LLM 驱动的自主 Agent 的潜力。AutoGPT 能够自主地将一个大目标拆解为子任务，逐步执行并根据结果调整策略，这在当时引起了极大的轰动。尽管这些早期项目在实际应用中还存在诸多局限——如容易陷入循环、成本高昂等——但它们开创了"LLM Agent"这一全新范式。

2023-2024 年：框架百花齐放 ​

随着 Agent 概念的火热，各大框架如雨后春笋般涌现。LangChain 通过其链式调用（Chain）和 Agent 模块成为最流行的 LLM 应用开发框架；CrewAI 专注于多 Agent 协作，让多个 Agent 以角色扮演的方式协同完成复杂任务；AutoGen（微软）则提出了多 Agent 对话的范式，支持灵活的 Agent 间通信模式。这一阶段的核心主题是探索 Agent 的架构设计和最佳实践。

2024-2025 年：走向生产 ​

Agent 技术开始从概念验证走向实际生产。Claude Computer Use（Anthropic）让模型能够直接操控计算机界面；Devin 号称"第一个 AI 软件工程师"，能够独立完成编程任务；SWE-agent 展示了 Agent 在真实 GitHub Issue 修复上的能力；Cursor 等 AI 编程助手将 Agent 能力深度集成到开发工作流中。这一阶段的特征是 Agent 从"Demo 级别"进化为"可用级别"。

当前阶段 ​

当前，AI Agent 正处于从实验室走向大规模应用的关键转折点。各大模型提供商（OpenAI、Anthropic、Google）纷纷推出原生的 Agent 能力和平台，Agent 的可靠性、安全性和可控性成为研究重点。多 Agent 协作、长期记忆、人机协同等方向持续演进。

Agent 核心架构 ​

根据 Lilian Weng 的经典博文，一个完整的 AI Agent 系统由以下核心组件构成：

Planning（规划） ​

规划能力是 Agent 将复杂任务转化为可执行步骤的关键。主要包括两个方面：

任务分解（Task Decomposition）：将一个复杂的、高层次的目标拆解为一系列更小、更易于管理的子任务。常用的技术包括：

  • Chain of Thought（CoT）：通过"让我们一步步思考"的提示，引导模型进行分步推理。
  • Tree of Thoughts（ToT）：在每一步探索多条推理路径，形成树状搜索结构，选择最优路径。
  • 任务分解提示：显式地要求模型将任务拆解为子任务列表，再逐一执行。

自我反思（Self-Reflection）：Agent 在执行过程中回顾自身行为，从错误中学习并改进策略。典型方法包括：

  • ReAct：将推理（Reasoning）和行动（Acting）交替进行，每次行动后观察结果并决定下一步。
  • Reflexion：引入反思机制，Agent 在任务完成后总结经验教训，存入记忆以供后续任务参考。
  • 自我批评：Agent 对自身的输出进行批判性评估，识别错误并修正。

Memory（记忆） ​

记忆系统使 Agent 能够在多次交互中保持上下文，积累知识和经验。参照人类记忆的分类方式，Agent 的记忆可分为：

  • 短期记忆（Short-term Memory）：对应 LLM 的上下文窗口（Context Window），存储当前对话或任务中的信息。受限于上下文长度，通常在数万 token 以内。
  • 长期记忆（Long-term Memory）：通过外部存储（如向量数据库）实现，可以持久化保存大量知识和经验。检索时通常使用相似性搜索（Similarity Search）从向量数据库中召回相关信息。
  • 感觉记忆（Sensory Memory）：对应多模态模型对视觉、听觉等输入的瞬时处理能力，如 GPT-4V 对图像的理解。

Tool Use（工具使用） ​

工具使用是 Agent 能力的核心延伸，使其不再局限于文本生成，而是能够与外部世界进行实际交互。主要包括：

  • Function Calling：模型根据用户请求自动选择合适的函数并构造参数，由系统执行后返回结果。OpenAI、Anthropic 等模型均已原生支持此能力。
  • API 集成：Agent 可以调用各种外部 API，如搜索引擎、数据库查询、文件系统操作等。
  • 代码执行：Agent 能够编写并执行代码来完成计算、数据处理等任务，这极大地扩展了其问题解决能力。

Agency Level（自主性等级） ​

在李宏毅教授的 AI Agent 课程中，引用了 HuggingFace 对 Agent 自主性等级的定义。理解这些等级有助于我们在实际应用中选择合适的 Agent 复杂度：

  • Level 0 — 简单处理器（Simple Processor）：没有任何自主性，模型仅对输入做出响应，相当于一个普通的 LLM 调用。例如：给定提示，生成一段文本。
  • Level 1 — 路由器（Router）：具备一定的选择能力，能够根据输入内容从预定义的选项中选择最合适的一个。例如：意图识别、情感分类等。
  • Level 2 — 工具使用（Tool Use）：能够根据任务需要调用外部工具和 API。Agent 可以判断何时需要使用工具、选择哪个工具、以及如何构造工具调用的参数。这是当前大多数 Agent 应用所处的等级。
  • Level 3 — 代码 Agent（Code Agent）：最高级别的自主性，Agent 能够编写、执行代码，甚至创建新的工具来解决问题。例如：Claude Code、Devin、SWE-agent 等编程 Agent。

在实际应用中，不是所有任务都需要最高级别的 Agent。选择合适的 Agency Level 需要在能力与可控性之间取得平衡。级别越高，Agent 的能力越强，但也越难以预测和控制。

Agent 的应用场景 ​

AI Agent 技术正在众多领域展现出巨大的应用潜力：

软件工程 ​

这是当前 Agent 应用最为成熟的领域之一。SWE-agent 能够自动修复 GitHub Issue；Devin 可以独立完成从需求分析到代码部署的全流程；Cursor 和 GitHub Copilot 则将 Agent 能力嵌入 IDE，提供实时的代码补全、重构和调试建议。编程 Agent 正在重新定义软件开发的方式。

科学研究与调研 ​

GPT Researcher 能够自主搜索互联网、阅读文献、整合信息并生成研究报告；AutoKaggle 面向数据科学竞赛，自动完成数据分析、特征工程和模型训练。在化学领域，ChemCrow 结合专业化学工具，辅助进行分子设计和反应预测。

计算机操作 ​

Mind2Web 和 WebArena 探索了 Agent 操控浏览器完成复杂网络任务的能力；Claude Computer Use 更进一步，使 Agent 能够直接操作桌面环境，包括鼠标点击、键盘输入和屏幕截图分析。

游戏与娱乐 ​

AI Agent 在游戏中的 NPC（非玩家角色）行为模拟方面展现出巨大潜力。通过 LLM 驱动，NPC 可以展现出更加自然、多样的行为模式和对话能力，极大提升游戏的沉浸感。

企业自动化 ​

在企业场景中，Agent 可以自动化处理客服工单、数据分析报告生成、邮件处理、日程安排等重复性工作，从而释放人力专注于更具创造性的任务。
